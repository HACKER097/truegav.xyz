No exploit. No malicious code. No bug in the contracts. Just wrong settings in a LayerZero DVN configuration.

The Kelp DAO event is the most expensive "oops" in recent DeFi history.

---

```text
Exploit:  none
Code bug: none
Settings: wrong
Loss:     millions
Fix:      one line in a config file
```

Every audit passed. Every test passed. The contracts were fine. The configuration was not.

DeFi doesn't fail inside contracts. It fails at the boundaries between them.

This time the boundary was a DVN misconfiguration on LayerZero. Next time it'll be something else. But the pattern is always the same: the smart contracts are fine; everything around them is on fire.
